# 套利路径详情 需求文档

增加新功能: 套利路径详情

点击套利路径的某个 机会 后, 会在当前页面中间出现弹窗.可以配置套利开始的代币买入数量, 默认是交易对设置的数量. 最后给出最终套利路径的代币数量.

用 机会 a 举例:

    (arbitrum) cbBTC -> WBTC  
    (ETH) WBTC -> CBBTC

点了上面的机会后,显示:

```
套利路径详情   [关闭]
[套利信息 ui]	[套利信息 ui]
[套利信息 ui]	[套利信息 ui]
```

其中,每个 [套利信息 ui] 的 ui 内容包括:

```
[初始投入金额输入框]
(arbitrum) cbBTC -> WBTC  [给出报价后的数量]
[显示报价用的 数据源]
[这里后续插入交易所链接,可以根据链和交易对自动拼接,后续实现,这次先不做]

(ETH) WBTC -> CBBTC  [给出报价后的数量]
[显示报价用的 数据源]
[这里后续插入交易所链接,可以根据链和交易对自动拼接,后续实现,这次先不做]

--------
收益  [] (收益率  [] )

```
提供 4 个 [套利信息 ui]  目的是比较不同的投入得到的产出,默认情况下: 

第一个的输入金额 是交易对里设置的数量
第二个: 0.2
第三个: 1
第4个:  2


为了配合实现这个功能, 方便”套利路径详情”上的报价更新的更快. 我希望能修改报价队列,  一旦 套利路径详情 页面激活了,  看板就暂停执行交易对的更新. 全力循环更新”套利路径详情” 上的报价

同样的, 报价看板 UI , 导航条的 保存配置 右边, 增加个 是否在报价中的状态 tag,  绿色报价中, 黄色暂停中

注意:

1. 这个功能需要新建个分支来实现
2. 快照回放中不需要这个功能, 只针对实时报价看板.
3. 代码尽量模块化,方便后续维护  先不要实现代码, 先和我确认功能和实现方式, 
4. 要用顶级程序员的眼光来解决问题, 也不要 ovewdesign, 不要引发新的 bug
