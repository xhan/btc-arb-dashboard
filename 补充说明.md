# 快照回放页

页面地址:
`/snapshot?time=2026-03-01 12:00:00`

说明:
- `time` 默认按 `UTC+8` 解析
- 页面是只读模式, 不会触发实时请求, 也不会改现有配置
- 可以用页面里的 `mode` 选择时间匹配策略:
  - `floor`: 取查询时间之前最近一条
  - `nearest`: 取前后更近的一条
  - `ceil`: 取查询时间之后第一条
- `maxGapSec` 可限制允许的最大时间偏差, 超过就不返回结果

页面功能:
- 查看特定时间点命中的快照报价
- 查看该快照下的套利路径
- 可直接点:
  - `查看快照 JSON`
  - `查看回放 JSON`

CLI:
`node scripts/replay-arb-snapshot.js --at "2026-03-01 12:00:00" --format text`

API:
- `/api/get-price-snapshot?at=2026-03-01%2012:00:00&mode=floor`
- `/api/replay-arb-snapshot?at=2026-03-01%2012:00:00&mode=floor&format=json`

# 看板价格波动箭头

- 箭头是前端基于当前报价和上一次报价的 `rawPrice` 比较得出的, 不是接口直接返回的状态。
- 仅在主报价成功刷新后判断, 入口在 `fetchSingleQuote -> updateTrendArrow`。
- 首次拿到价格时, 因为没有旧价格, 不显示箭头。
- 如果新旧价格相等, 不显示箭头。
- 如果本次成功报价源和上次成功报价源不同, 不显示箭头, 避免因为 source fallback / source 切换造成误判。
- 只有相对波动幅度 `abs((currentPrice - oldPrice) / oldPrice) >= 0.0001` 时才显示, 也就是至少约 `0.01%`。
- `currentPrice > oldPrice` 显示上涨箭头 `↑↑↑`; `currentPrice < oldPrice` 显示下跌箭头 `↓↓↓`。
- 箭头显示 30 秒后自动隐藏; 如果这 30 秒内又有新波动, 会重置计时并按最新方向显示。


# 看板报警逻辑

- 报警是在每次拿到新报价后立即判断, 没有“连续命中几次才触发”的逻辑。
- 只要某一次最新报价满足报警条件, 就会触发报警。
- 百分比报警比较的是 `rawPrice` 和 `basePrice`:
  - 上涨报警: `((newRawPrice - basePrice) / basePrice) * 100 >= percentUp`
  - 下跌报警: `((newRawPrice - basePrice) / basePrice) * 100 <= -percentDown`
- 目标价报警比较的是 `finalAmountOut`:
  - 高于目标: `newTotalAmountOut >= targetAbove`
  - 低于目标: `newTotalAmountOut <= targetBelow`
- 这意味着:
  - 百分比报警看的是单价 / 汇率。
  - 高于 / 低于目标价看的是当前输入数量对应的总输出值, 不一定等于单价。
  - 对 Bybit, `finalAmountOut` 就是价格本身; 对链上报价, `finalAmountOut` 一般是当前 `amount` 对应的输出总量。

## 基准价 `basePrice`

- 只有设置了“上涨超过”或“下跌超过”时, 才会使用 `basePrice`。
- 保存报警设置时:
  - 如果勾选“更新基准价格为当前价格”, 会用当前最新 `lastRawPrice` 重置 `basePrice`。
  - 如果没勾选, 且之前已有 `basePrice`, 就沿用旧值。
  - 如果当时还没有拿到当前价格, `basePrice` 可能不会写入, 这时百分比报警不会生效, 直到下次重新保存出有效基准价。

## 触发后的表现

- 命中条件后, 当前报价卡片会加高亮。
- 声音提醒会进入循环播放状态。
- 提醒日志会新增一条记录。
- 同一轮持续命中条件时, 不会每次刷新都追加新日志, 因为内部用 `logShown` 避免重复记日志。
- 只有后续某次报价不再满足条件时, `logShown` 才会重置; 下次再次命中时才会重新写一条日志。
- 点 `✔️` 只会清除未读状态 / 高亮 / 声音, 不会删除报警条件本身。
