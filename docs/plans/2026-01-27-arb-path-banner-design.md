# 套利路径展示设计（WBTC监控）

日期：2026-01-27

## 背景与目标
在现有“聚合报价看板 3.0”基础上，增加“套利路径”窗口，实时展示 **WBTC监控** 分区内最大价差路径。路径计算允许跨链同名资产直连（不计桥费），先实现双腿闭环（2-step）路径。

## 范围
- 仅改前端 `index.html`。
- 仅使用 `WBTC监控` 分区内报价。
- 仅展示当前最大价差路径一条。
- 先支持 cbBTC ↔ WBTC 这类 2-step 闭环。

## 关键数据
- `dashboardState`：配置与分区/报价列表。
- `quoteMonitorState`：每条报价的最新价格与元数据（新增反向报价缓存）。

## 路径模型
- 报价被视为有向边：`fromSymbol -> toSymbol`，附带 `chain` 与 `rate`。
- 若 `showInverse` 且成功获取反向报价，生成反向边。
- 路径计算按 **symbol** 连接（忽略链，链仅用于展示）。
- 双腿闭环收益率：`rate1 * rate2 - 1`。
- 选择收益率最高的路径作为展示结果。

## 数据流
1. `fetchSingleQuote` 成功后更新 `quoteMonitorState`。
2. 同步保存反向报价（如有）。
3. 触发 `updateArbPanel()`（轻量节流，约 300ms）。
4. 计算最大收益路径并渲染到“套利路径”窗口。

## UI 展示
- 新增窗口“套利路径”，复用“提醒日志”样式。
- 位置：左下角（`left: 20px; bottom: 20px;`）。
- 默认宽度 480px 左右，支持 resize。
- 内容示例：
  - `cbBTC(ETH) -> WBTC(ETH) @1.0002`
  - `WBTC(Arb) -> cbBTC(Arb) @1.0005`
  - `循环收益: +0.07%`
- 若缺少数据，显示“等待数据 / 暂无可用路径”。

## 错误处理
- 若任一边报价缺失或失败，不展示旧路径，标记“数据不完整”。

## 后续扩展
- 扩展多跳路径（如 cbBTC -> tBTC -> WBTC）。
- 加入桥费/手续费模型。
- 支持多分区路径分析。
