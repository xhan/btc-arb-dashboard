# 套利路径多腿扩展设计（WBTC/LBTC/TBTC）

日期：2026-01-27

## 目标
将套利路径从单分区、两腿闭环，扩展为三个分区（WBTC监控/LBTC监控/TBTC监控）参与的多腿闭环（最多 4 腿），并加入规则：xBTC 与 cbBTC 等价（汇率固定为 1）。

## 范围
- 仅改前端逻辑（`index.html` + `arb-paths.js`）。
- 路径计算来源：WBTC监控、LBTC监控、TBTC监控。
- 允许 2～4 腿闭环；展示最大收益路径。
- 规则边：xBTC <-> cbBTC，汇率恒为 1。

## 数据与汇率说明
- 单位汇率使用 `raw_price = amountOut / amount`，自动折算。
- 汇率反映该数量的真实滑点，是实际成交报价的单位价。

## 路径模型
- 报价生成有向边：`fromSymbol -> toSymbol`。
- 若 `showInverse` 且反向成功，追加反向边。
- 注入规则边：`xBTC -> cbBTC` 与 `cbBTC -> xBTC`，链标记为“规则”。

## 算法（最多 4 腿闭环）
- 对每个起点 symbol 做 DFS。
- 路径中不重复访问 symbol（允许最后回到起点）。
- 深度上限 4；遇到起点闭环即计算收益：`product(rate) - 1`。
- 选收益率最大的路径。

## UI 展示
- 每一腿一行：`（链）from -> to @rate`。
- 最后一行显示总收益率，正负用不同颜色。
- 无有效闭环时显示“暂无可用路径 / 等待数据”。

## 异常处理
- 若 `ArbPaths` 未加载，显示“路径模块未加载”。
- 报价缺失时不使用旧路径，避免误导。

## 测试
- 单元测试覆盖：
  - DFS 多腿闭环（3/4 腿）
  - xBTC=cbBTC 规则边参与路径
  - 展示格式
- UI 继续人工验证。
